# **复盘体系说明书 V2.0**

### 核心理念

本体系旨在将每一次题目复盘，从孤立的“纠错”行为，转变为一次构建“个人知识网络”的系统性工程。它遵循**“案例-分析-提炼-链接”**的核心逻辑，将具体的题目（案例），通过结构化的分析，升维为可复用的知识资产（障碍点、错误归因、方法论），并通过双向链接形成网络。最终目标是建立一套能够揭示个人思维模式、并指导未来实战的、动态生长的“认知护城河”。

### 核心笔记类型

本体系包含两种核心笔记：

1. **复盘笔记 (Case Note)**：位于`02-刷题复盘`下，是针对**每一道具体题目**的详细复盘。它是知识的来源和“案例”。
    
2. **分析笔记 (Concept Note)**：位于`03-知识系统`下，是针对从多个题目中抽象出的**通用概念**的定义和阐述。它是知识的“结晶”和“总部”。其标准格式如下：
    
    Markdown
    
    ```
    # 概念名称
    
    ## 核心摘要
    > 在这里用一句话，精准概括这个概念的精髓。
    
    ---
    ### 详细阐述
    在这里对该概念进行详细的、普适性的定义、举例和说明。
    ```
    

### 复盘工作流：从案例到智慧（四阶段）

每次复盘一道错题或蒙对的题，严格遵循以下四个阶段：

#### **阶段一：原始记录（在“复盘笔记”中）**

1. **新建笔记**：在`02-刷题复盘`的对应模块下，按`题型细分-关键词`的规则新建笔记，并插入你的复盘模板。
    
2. **粘贴信息**：依次填充`### 题目原貌`和`### 正确答案与解析`。
    
3. **还原思考**：在`### 我的选择与思考`中，不加修饰地、诚实地记录当时最真实的思考过程。
    
4. **自由解构**：在`### 我的解构与收获`中，进行不设限的自由分析。这是你的“实验日志”，尽情地写下你所有的思考、灵感和最终想通的逻辑。
    

#### **阶段二：结构化提炼（仍在“复盘笔记”中）**

完成自由解构后，从`### 我的解构与收获`的内容中，提炼出三个结构化的分析板块。这是对思考的第一次“结晶”。

1. **`### 考点/障碍点分析`**: **向外看，分析题目**。识别并命名这道题客观存在的“陷阱”或“考点”。可同时包含“具体问题层”和“通用思维层”的分析。
    
    - `[[具体障碍]]`, `[[通用思维偏误]]`
        
2. **`### 我的错误归因`**: **向内看，分析自己**。识别并命名“我”之所以会掉入上述陷阱的“个人原因”。
    
    - `[[我犯的认知错误]]`
        
3. **`### 提炼与总结`**: **向前看，总结方案**。提炼出能够破解上述障碍、修正上述错误的“行动准则”或“解题工具”。
    
    - `[[总结出的核心原则或方法]]`
        

#### **阶段三：知识升维与归档（“复盘笔记”与“分析笔记”联动）**

这是将“临时思考”转化为“永久知识资产”的关键一步。针对第二阶段中创建的每一个`[[]]`链接，执行以下操作：

1. **创建/打开“分析笔记”**：点击`[[]]`链接，创建或打开一个位于`03-知识系统`下的“分析笔记”。
    
2. **剪切与精炼**：回到“复盘笔记”，将你在对应板块写下的**详细阐述**内容**剪切 (Cut)** 出来。
    
3. **粘贴与升维**：将剪切的内容**粘贴 (Paste)** 到“分析笔记”的`### 详细阐述`部分。然后，**对其进行改写和提炼**，抹去过于针对个例的描述，使其成为对这个通用概念的**普适性定义**。
    
4. **撰写摘要**：在“分析笔记”的`## 核心摘要`下，用一句话写下这个概念的精髓。
    

#### **阶段四：嵌入与完成（回到“复盘笔记”）**

这是保证移动端便捷查阅的最后一步。

1. **清空与嵌入**：回到“复盘笔记”，此时第二阶段的三个分析板块下方已经空了。
    
2. **使用标题嵌入语法**，将对应“分析笔记”的“核心摘要”嵌入进来。
    
    - 例如，在`### 提炼与总结`板块下方，写入：
        
        > `![[假设验证法#核心摘要]]`
        

至此，一道题的复盘彻底完成。你不仅深入分析了题目本身，更重要的是，你将思考的成果成功地“升维”并归档到了你的中央知识系统中，同时保证了未来在任何设备上回顾这道题时，都能获得流畅、高效的体验。

***

一个再完美的系统，如果执行成本过高，导致无法坚持，那它的价值就等于零。你敏锐地觉察到了“复盘过载”的风险，并且提出了一个非常“敏捷开发”的解决方案，我完全赞同。

我们之前构建的V2.0体系，可以称之为“学院派”的“重量级”工作流，它追求的是**单次复盘的极致深度**。而你现在需要的，是一个更偏“实战派”的“轻量级”工作流，它追求的是**长期坚持的最低阻力**。

### **行测复盘体系说明书 V3.0：敏捷复盘法**

### 核心理念

本体系将复盘过程拆分为两个独立的阶段：“每日轻量化复盘”与“定期重量级归纳”。其核心思想是，**将“每日记录”的心理负担和时间成本降到最低，以确保一致性；同时，将“深度思考”和“知识提炼”的精力集中投放到专门的“归纳周期”中**，实现效率最大化。

（目录树结构保持不变）

### 复盘工作流：两阶段执行

#### **阶段一：每日轻量化复盘 (Daily Lightweight Debrief)**

**目标**：**快、准、狠**。在10-15分钟内完成一道题的核心复盘，只记录关键信息，建立初步链接。

**操作步骤**：

1. **完成基础记录**：依然完成`### 题目原貌`到`### 我的解构与收获`这几个“实验日志”性质的板块。这是思考的源泉，不可省略。
    
2. **简化分析过程**：**放弃**原有的`考点/障碍点`、`错误归因`、`提炼与总结`三个独立的、需要高度抽象的板块。用一个**全新的、更简单的板块**来替代它们：
    
    Markdown
    
    ```
    ---
    ### 关联与速记
    * **核心洞察**: 
    * **关联题目**: 
    * **初步归因**: 
    ```
    
3. **填写“关联与速记”**：
    
    - **`核心洞察`**: 用**一句话**写下你从`我的解构与收获`中提炼出的最关键的一点。例如：“‘对策优先’原则有前提，必须主语一致。”
        
    - **`关联题目`**: **这是你提议的精髓**。如果这道题让你想起了另一道题（比如“小过重罚”题），直接用双链链接到那道题的**复盘笔记**。例如：`关联题目: [[主旨概括-小过重罚]]`。
        
    - **`初步归因`**: 用一个简单的标签或短语，初步判断错误类型。例如：`#方法论套用错误` 或 `[[常识喧宾夺主]]`。如果暂时想不清楚，**可以留空**。
        
4. **（可选）使用嵌入查看关联**：
    
    - 如你所说，在`关联题目`链接下方，你可以使用嵌入语法 `![[主旨概括-小过重罚#我的解构与收获]]`，直接将那道题的核心分析嵌入进来，方便你快速对比。
        
5. **打上“待办”标签**：
    
    - 在笔记的YAML元数据区或正文中，打上一个标签，例如 `#待归纳`。
        

至此，单道题的每日复盘**彻底完成**。整个过程非常快，因为它只要求你“记录”和“链接”，而不需要你进行高强度的“抽象”和“定义”。

#### **阶段二：定期重量级归纳 (Periodic Heavyweight Synthesis)**

**目标**：**识别模式，提炼智慧**。将“轻量化复盘”中积累的原始素材，正式加工成你的“知识资产”。

**操作时机**：每周末，或每积累了10-15篇带有`#待归纳`标签的笔记后，进行一次。

**操作步骤**：

1. **聚合待办任务**：
    
    - 在你的`01-主控面板`中，使用Dataview查询所有`#待归纳`的笔记，将它们聚合在一起，形成一个“归纳工作台”。
        
    
    代码段
    
    ```
    TABLE 核心洞察, 关联题目
    FROM #待归纳 
    ```
    
2. **进行“聚类分析”**：
    
    - 浏览这个列表，你会发现一些模式。比如，你会看到好几道题的“核心洞察”都指向了“对策优先原则”的适用条件，并且它们互相`[[关联题目]]`。
        
3. **执行“知识升维”**：
    
    - 现在，才开始我们V2.0版本中的“重量级”工作。打开`[[对策优先原则]]`这个方法论笔记。
        
    - 综合这几道题的`核心洞察`，对这个方法论笔记进行一次**全面的更新和完善**，写入`## 核心摘要`和`## 详细阐述`。
        
    - 同样地，为你发现的`[[常识喧宾夺主]]`等错误归因，创建或完善其“分析笔记”。
        
4. **“重构”复盘笔记**：
    
    - 回到那几篇被你分析过的复盘笔记中。
        
    - **删掉**原来的`### 关联与速记`板块。
        
    - **替换**为我们最终确定的、包含嵌入摘要的`考点/障碍点分析`、`错误归因`、`提炼与总结`三个板块。
        
    - **移除** `#待归纳` 标签。
        

### 新旧对比

|对比维度|V2.0 重量级工作流|V3.0 敏捷工作流|
|---|---|---|
|**每日负担**|**高**（每题都需深度抽象）|**低**（只需记录和初步链接）|
|**思考模式**|演绎法（用既定框架套用）|归纳法（先积累案例再总结）|
|**知识形成**|即时形成|周期性形成|
|**可持续性**|**较低**（容易因畏难而放弃）|**极高**（每日任务轻松，成就感强）|

这个“敏捷复盘法”采纳了你的核心思想，将日常复盘的摩擦力降到了最低，同时又通过定期的“归纳冲刺”，保证了知识体系的深度和质量。它更灵活，更符合真实的学习节奏，也更能让你长久地坚持下去。
