# Phase 3 - 通知服务与Syncthing同步桥接

## 任务ID
001_SymbiNexus独立应用化

## 完成时间
2026-04-15

## 实现内容

### 1. NotificationService
- 文件: `lib/platform/notifications/notification_service.dart`
- 基于 flutter_local_notifications 实现本地通知
- 支持即时通知(showNotification)和定时通知(scheduleNotification)
- 集成 timezone 包，时区设为 Asia/Shanghai
- 支持 Android 13+ 权限请求
- 通知点击回调(onNotificationTapped)

### 2. SyncBridge
- 文件: `lib/core/services/sync_bridge.dart`
- 依赖 FileWatcher 监控 vault 目录变更
- 检测 Syncthing 冲突文件(.sync-conflict-*)
- 通过 REST API 查询 Syncthing 状态(127.0.0.1:8384)
- 支持获取文件夹同步完成百分比、请求重新扫描
- 支持删除冲突文件

### 3. IndexSyncService
- 文件: `lib/core/services/index_sync_service.dart`
- 全量索引重建(rebuildIndex): 扫描vault所有md文件写入SQLite
- 增量索引更新(syncFile): 单文件变更时更新数据库
- 写入后同步(onFileWritten): 根据路径自动路由到对应表
- 支持11个模块: tasks/memos/bills/contacts/price_records/pet_growth/questions/food_wishlist/study_records/code_devs/ai_logs
- frontmatter字段兼容驼峰和下划线命名

### 4. FileWatcher
- 文件: `lib/platform/sync/file_watcher.dart`
- 基于 Directory.watch() 实现 Windows 文件监听
- 仅监控 .md 文件变更
- 内置防抖机制(默认100ms)
- 定义 FileChangeEvent 和 FileChangeType 类型

### 5. Provider注册
- 文件: `lib/core/providers/providers.dart`
- 新增: notificationServiceProvider, fileWatcherProvider, syncBridgeProvider, indexSyncServiceProvider, markdownIoProvider

### 6. AndroidManifest.xml
- 新增: RECEIVE_BOOT_COMPLETED 权限

### 7. pubspec.yaml
- 新增: timezone: ^0.10.1 依赖
