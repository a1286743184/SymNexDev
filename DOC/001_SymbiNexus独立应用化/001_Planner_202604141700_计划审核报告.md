# SymNex 实施计划审核报告

> Planner | 2026-04-14 | 审核对象: 001_Planner_202604141600_SymNex详细实施计划.md

---

## 一、总体评价

计划结构完整、任务粒度合理、验收标准明确，是一份可执行的高质量实施计划。但在**工期估算**和**关键路径分析**上存在系统性偏差，主要体现在：

1. **新手系数不足**: 对Dart/Flutter新手的探索性学习时间估算偏低，实际效率约为熟手的40-60%
2. **关键路径计算错误**: 计划声明的关键路径包含P2-05和P3-01，但这两项并非关键路径节点
3. **并行机会未利用**: 多个仅依赖P1-04的任务(P3-03, P4-04)被推迟到后续Phase，浪费了并行窗口

**审核结论**: 计划需修订后再执行。乐观工期应调整为20周，最可能工期24周，悲观工期30周。

## 三、关键路径分析

### 计划声明的关键路径(有误)

```
P0-01 -> P0-02 -> P0-03 -> P0-04 -> P1-01 -> P1-02 -> P1-04 -> P2-01 -> P2-02 -> P2-05 -> P3-01 -> P4-05 -> P5-01 -> P5-02
```

### 实际关键路径(修正)

```
P0-01 -> P0-02 -> P0-03 -> P0-04 -> P1-01 -> P1-02 -> P1-04 -> P1-06 -> P2-01 -> P2-02 -> P4-05 -> P5-01 -> P5-02
```

**修正说明**:

1. **P1-06应在关键路径上**: P2-01依赖P1-06(Provider体系)，而P1-06依赖P1-02+P1-03+P1-04，因此P1-06是P2-01的前置条件。

2. **P2-05不在关键路径上**: P2-05(数据迁移)依赖P2-02+P2-03+P2-04，但P2-05本身没有下游依赖。它是一个"收尾"任务，不在关键路径上。

3. **P3-01不在关键路径上**: P3-01(通知系统)依赖P2-03+P2-04，但P3-01没有关键路径上的下游依赖。

4. **P4-05在关键路径上**: P4-05(性能优化)依赖P2-02+P2-03，而P5-01依赖P4-05，因此P4-05在关键路径上。

## 五、优化建议

### 5.1 缩短关键路径

**建议1: AI管道分期交付 (节省2-3周)**

P2-01是关键路径上的最大节点。建议拆分为两期:

- **MVP期**: 仅支持OpenAI兼容协议(覆盖6/8供应商: 智谱/千问/谷歌/豆包/阿里/美团)，跳过Anthropic协议(稀宇/小米)
- **增强期**: 补充Anthropic协议支持

这样P2-01a从5-6天降至3-4天，且减少了一半的调试工作量。

**建议2: P1-04 Markdown解析分层 (节省1周)**

当前P1-04试图一次性解决所有md格式解析。建议分层:

- **MVP层**: 仅支持账单md(frontmatter格式统一) + 备忘录(checkbox格式)
- **增强层**: 角色档案(YAML数组) + Dataview内联字段 + `%%`折叠块 + 宠物档案

这样P1-04从20-28h降至12-16h，且降低了R-01风险的即时影响。

**建议3: 提前启动P3-03 (节省1周)**

P3-03(Syncthing同步)仅依赖P1-04，可在Phase 2期间与P2-01并行启动。这是最高风险任务之一，越早开始越能暴露问题。

### 5.2 可简化或延后的任务

| 任务 | 建议 | 节省工期 |
|------|------|---------|
| P2-05 数据迁移 | 延后到Phase 4之后，MVP使用手动导入或直接读取md | 1.5周 |
| P3-02 快速录入Widget | 延后到v1.1，MVP通过通知栏快捷操作替代 | 3天 |
| P4-01~P4-03 P1功能模块 | 延后到v1.1，MVP仅保留AI管道+账务+任务+备忘 | 9天 |
| P4-04 Templater替代 | 延后到v1.1，MVP不做自动模板 | 3天 |
| P2-01c 离线队列 | 简化为仅本地缓存，不做自动重试 | 1天 |
| P5-04 用户文档 | 仅在用户要求时编写 | 2天 |

### 5.3 MVP精简方案

**核心思路**: 聚焦"AI录入+账务+任务"铁三角，确保核心闭环可用。

```
MVP范围:
  Phase 0: 环境搭建 (4天)
  Phase 1: 基础框架 (2.5周) -- 精简P1-04
  Phase 2-core:
    P2-01 AI管道 (2.5周) -- 仅OpenAI协议
    P2-02 账务收支 (2周)
    P2-03 任务管理 (2周)
    P2-04 备忘提醒 (1周)
  Phase 5-lite: 基础测试+签名 (1周)

MVP总工期: ~9-10周

v1.1范围:
  P2-05 数据迁移
  P3-01~P3-04 平台集成
  P4-01~P4-05 增强体验
  Anthropic协议支持
  完整frontmatter解析

v1.1增量工期: ~8-10周
```

**MVP优势**:
1. 10周内可交付可用产品，获得早期反馈
2. 核心闭环(AI录入->账单/任务/备忘)可验证
3. 降低R-01(frontmatter)和R-02(Syncthing)的即时风险
4. 新手学习曲线分散到更长时间

**MVP劣势**:
1. 没有Syncthing同步，数据仅在本地
2. 没有通知系统，备忘提醒不可用
3. 没有数据迁移，需手动操作

### 5.4 其他建议

1. **建立性能基准**: 在Phase 1结束时建立冷启动、列表滚动等基准数据，避免Phase 4性能优化时无参照
2. **CI/CD尽早配置**: 在P0-04完成后配置GitHub Actions自动运行flutter analyze和flutter test
3. **真实数据测试集**: 从现有SymbiNexus库中提取10-20个典型md文件作为固定测试集，避免后期才发现解析问题
4. **周度里程碑检查**: 每周五检查关键路径进度，偏差>2天时触发计划修订
