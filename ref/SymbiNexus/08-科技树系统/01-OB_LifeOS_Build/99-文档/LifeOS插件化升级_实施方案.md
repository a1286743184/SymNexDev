# LifeOS 插件化升级实施方案

## 1. 总体方案
- 以“LifeOS Engine 插件”作为唯一运行时与配置中枢。
- 任务与财务为两个内置模块，共享统一的渲染与配置框架。
- md 看板与配置页保留为“壳文件”，仅负责调用插件渲染。
- 采用插件 CodeBlock 作为渲染入口，移除 DataviewJS 依赖。

## 2. 交付结构
```
.obsidian/plugins/lifeos-engine/
  manifest.json
  main.js
  styles.css
  assets/echarts.min.js
  data.json
```
- 插件内置 Finance 与 Task 引擎。
- 所有渲染与配置读写均从插件目录加载。

## 3. 关键技术设计
### 3.1 渲染入口
- 注册 Markdown CodeBlock：
  - lifeos-task
  - lifeos-finance
  - lifeos-config
- md 壳示例：
```
```lifeos-finance
mode: month
period: 2026-01
```
```

### 3.2 配置管理
- data.json 作为唯一来源。
- 首次启动自动导入旧配置文件并迁移。
- 插件设置页分为“任务/财务”两个 Tab。
- 提供导入/导出与备份入口。

### 3.3 控制台（Console）
- 以插件面板/弹窗实现主要入口。
- 同时提供 md 壳控制台作为可选入口。
- 控制台支持：
  - 快捷命令打开
  - 关键操作按钮（创建看板、索引管理、配置入口）

### 3.4 任务引擎
- 扫描 sources 配置的目录/文件。
- 解析 `- [ ] / - [x]` 任务行与现有日期时间标记。
- 建立任务索引缓存，支持增量刷新。

### 3.5 财务引擎
- 兼容现有 frontmatter 账单与 CSV/ledger。
- 6 分区配置在插件内完整实现。
- ECharts 从插件 assets 加载或内置。
- 缓存索引机制由插件统一管理。

### 3.6 文件路径与联动
- 看板文件生成目录可配置默认路径。
- 记录文件名与路径映射，路径失效时按文件名重匹配并更新配置。

## 4. 迁移步骤
1. 创建插件骨架与运行时入口。
2. 内置 Finance 引擎并完成渲染、索引、配置迁移。
3. 内置 Task 引擎并完成任务扫描与渲染。
4. 批量替换库内 md 看板为 CodeBlock 壳。
5. 创建控制台与配置 md 壳（可选）。
6. 完成回归验证与性能检查。
7. 输出旧文件清理清单并由用户确认删除。

## 5. 更新机制
- 提供离线更新脚本（bat + ps1）：
  - 输入 vault 路径，定位插件目录
  - 备份旧版本
  - 白名单覆盖版本文件
  - 保留 data.json 与用户配置
- 输出更新日志与回滚路径。

## 6. 验证与回归
- 任务：今日聚焦数据一致、换周归档逻辑一致、来源/标签展示一致。
- 财务：月度/年度统计一致、趋势/TopN/预算一致、缓存文件生成正常。
- 配置：插件设置页与 md 配置页均可正常使用。
- 稳定性：移动文件后能按文件名重绑定并继续渲染。

## 7. 风险与控制
- 兼容性风险：通过自动迁移与配置回退机制控制。
- 性能风险：通过索引缓存、最大扫描数与增量更新控制。
- 更新风险：通过备份与白名单覆盖控制。
