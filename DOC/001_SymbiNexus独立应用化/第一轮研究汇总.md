# SymbiNexus 独立应用化 -- 第一轮研究汇总

> 汇总时间: 2026-04-14 | 基于4个子代理的第一轮研究结果

---

## 一、现有系统架构理解（已确认）

### 核心架构
- **管道式数据流** + **模块化路由** + **壳文件渲染** 三层架构
- **唯一入口**: smart-input-pro 插件，两阶段AI管道（分类 -> 结构化提取）
- **8大子系统**: 经纬矩阵/复盘/成就/模板/生活坐标/财务/项目/科技树
- **11个AI分类模块**: bill/task/memo/contact/question_entry/code_dev/food_wishlist/price_tracker/study_record/pet_growth/other
- **8个LLM提供商**: 智谱/稀宇/千问/小米/谷歌/豆包/阿里/美团LongCat
- **插件协作链**: SIP(写入) -> LifeOS Engine(读取渲染) -> View Controller(视图) -> Command Orb(快捷) -> Dataview(查询)

### 关键设计模式
- 3点枢轴偏移: 解决凌晨工作归属问题
- 渠道强制归一: AI输出后二次校验
- 壳文件模式: md文件标记lifeos.kind，代码块处理器渲染
- 降级容错: AI失败自动降级到INBOX

## 二、技术选型初步方案

### 核心决策对比

| 决策项 | 架构师推荐 | 开发环境研究推荐 | 冲突点 |
|--------|-----------|----------------|--------|
| 跨平台框架 | **Flutter 3.41+** | **Tauri 2.0** | 需要交叉审核 |
| C盘占用 | ~200-500MB | ~100-300MB | Tauri更优 |
| Android原生能力 | home_widget/workmanager成熟 | tauri-plugin-widgets可用 | Flutter更成熟 |
| 开发效率 | Dart上手快，Hot Reload | Rust门槛高，已有环境 | Flutter更优 |
| 看板渲染 | Skia自绘，质量高 | WebView渲染 | Flutter更优 |
| 已有环境 | 需全新搭建 | Rust/Tauri已在D盘 | Tauri更优 |

### 已达成共识的决策

| 决策项 | 共识方案 |
|--------|---------|
| 数据存储 | SQLite/drift + Markdown双层（文件为真相源） |
| 全文搜索 | SQLite FTS5 + jieba中文分词 |
| AI管道 | 3阶段LLM管道 + 8提供商路由 + 规则引擎降级 |
| 通知系统 | WorkManager + flutter_local_notification / tauri-plugin-notification |
| 桌面小组件 | home_widget / tauri-plugin-widgets |
| Syncthing集成 | 外部App + REST API（MVP阶段） |
| 数据迁移 | 渐进式：MVP用Markdown优先，正式版DB优先+md双写 |

## 三、功能迁移完整性（已确认）

### 迁移分类统计
- **可直接迁移**: AI管道提示词、API调用逻辑、colored-tags/regex-mark等辅助功能
- **需重新设计**: 账单存储、任务看板、财务看板、备忘系统（增加通知）
- **可显著增强**: 通知提醒、桌面小组件、价格趋势图、宠物体重曲线、刷题正确率

### MVP范围（已确认）
- P0必须: AI智能录入管道 + 账务收支 + 任务管理 + 备忘提醒 + Markdown读写 + 数据迁移工具
- P1应该: 价格追踪 + 宠物成长 + 角色档案 + Android通知 + 快速录入Widget
- P2可以: 财务看板完整版 + 美食收藏 + 刷题记录 + 题目录入
- MVP预估工期: 10-14周（单人开发）

## 四、关键冲突与待决策项

### 冲突1: Flutter vs Tauri（核心决策）

**Flutter优势**:
1. Android原生能力插件最成熟（home_widget/workmanager/flutter_local_notification）
2. Skia自绘引擎，记账看板/饼图/趋势图渲染质量远优于WebView
3. 生态4万+包，大厂验证
4. Dart比Rust上手快，单人开发效率高
5. Hot Reload <200ms重载

**Tauri优势**:
1. C盘占用最小（~100-300MB vs Flutter ~200-500MB）
2. 用户已有完整Rust/Tauri开发环境在D盘
3. 包体积极小（2-5MB vs Flutter 8-15MB）
4. 内存占用低（20-60MB vs Flutter 60-120MB）
5. Windows端原生优势（系统WebView2）

**待审核问题**:
- Tauri 2.0的Android端生态是否足够成熟？
- tauri-plugin-widgets的稳定性和功能覆盖是否满足需求？
- WebView渲染对复杂看板（饼图/趋势图/热力图）的性能是否可接受？
- 单人开发Rust后端的效率损失有多大？

### 冲突2: 数据存储策略细节

- MVP阶段Markdown优先 vs DB优先的选择时机
- 双写一致性的保障机制
- 首次启动索引重建的性能（2000文件10-30秒是否可接受）

### 冲突3: 开发环境搭建顺序

- Flutter需要全新搭建（Android SDK + Flutter SDK + JDK 17）
- Tauri已有环境但需补充Android SDK
- 两者都需要Android SDK，可以共享
