3点枢轴逻辑模块使用说明（DataviewJS / Templater 集成）

目标
- 提供统一入口的“3 点枢轴”日期逻辑（默认枢轴小时为 3），把本地时间 0:00–2:59 视为上一“逻辑日”。
- 在页面内通过 dynbedded 引入一次，即可复用所有便捷函数（today、completion、逾期判断等）。
- 重要行为修正：对“仅日期”值（如 YYYY-MM-DD 或本地 00:00:00）不应用枢轴偏移，视为该自然日，避免被错误前移一天。

模块位置
- 代码文件：04-模板系统/00-核心模块/枢轴偏移.dvjs.md
- 导出对象：window.ObLife.pivot3am（下文简称 P）
- 默认枢轴小时：P.DEFAULT_PIVOT_HOUR = 3

如何引入（dynbedded 单入口）
- 在页面开头加入：
  ```dynbedded
  [[枢轴偏移.dvjs]]
  ```
- 在 DataviewJS 中使用（两种方式任选其一）：
  1) 直接使用对象方法：
     ```dataviewjs
     const P = window.ObLife?.pivot3am;
     const todayISO = P.logicalTodayISO(3);
     ```
  2) 使用绑定便捷函数（推荐）：
     ```dataviewjs
     const { todayISO, completionISO, isLateByPivot } = window.ObLife.pivot3am.bind(3);
     ```

核心行为与函数
- 本地时区统一化：所有输入（字符串/Date/Luxon）会先转换为本地时区，再进行枢轴计算，避免 UTC 字符串在本地被误判为凌晨而前移。
- “仅日期”不偏移：当输入为纯日期（YYYY-MM-DD）或本地 00:00:00 时，视为自然日，不应用 3 点枢轴前移。
- today（逻辑今天）：
  - todayISO = P.logicalTodayISO(3)
  - 返回逻辑日（枢轴后）的 ISO 日期字符串（YYYY-MM-DD）。
- 完成日桥接（Dataview 任务的完成时间 → 逻辑日期）：
  - P.completionISO(t.completion, 3) → 完成的“逻辑 ISO 日期”
  - P.completionDT(t.completion, 3) → 完成的“逻辑 Luxon DateTime（startOf('day')）”
  - 说明：若 t.completion 为“仅日期”，不做前移；若为具体时间，只有在本地时间 < 03:00 时才视为上一逻辑日。
- 逾期判断（枢轴逻辑下的到期 vs 完成）：
  - P.isLateByPivot(t.due, t.completion, 3) → true/false
  - 实现要点：比较的是枢轴后的“ISO 日期字符串”（dueISO vs compISO），而不是具体时间点；保证“同日到期且同日完成”不会被误判为逾期。
- 逻辑日归属判断：
  - P.belongsToLogicalDate(dt, isoDate, 3) → 某时间是否属于指定逻辑 ISO 日期。
- 逻辑日边界（用于事件过滤等）：
  - P.startOfLogicalDay(baseDT, 3) / P.endOfLogicalDay(baseDT, 3)
- 取消标识检测（文本型标记）：
  - P.isCancelledLine(text) → 是否匹配“❌ YYYY-MM-DD”。

快速示例
- 今日聚焦（按枢轴计算“今天”）：
  ```dataviewjs
  const { todayISO, completionISO, isLateByPivot } = window.ObLife.pivot3am.bind(3);
  const L = dv.luxon;
  const today = L.DateTime.fromISO(todayISO).startOf('day');
  // 分类示例（胜利结算/为时不晚/妙手偶得）可直接使用 completionISO / isLateByPivot
  ```
- 本周聚焦（枢轴今天 → 推导本周范围）：
  ```dataviewjs
  const { todayISO, completionISO, isLateByPivot } = window.ObLife.pivot3am.bind(3);
  const L = dv.luxon;
  const today = L.DateTime.fromISO(todayISO).startOf('day');
  // 用 today 推导 weekStart/End，再用 toISODate 得到 weekStartISO/EndISO
  // 完成在本周且不逾期：t.completed && compISO ∈ [weekStartISO, weekEndISO] && isPlanned(t) && !isLateByPivot(t.due, t.completion)
  // 完成但逾期：t.completed && compISO ∈ [weekStartISO, weekEndISO] && t.due && isLateByPivot(...)
  // 无计划但完成：t.completed && !t.due && !t.scheduled && compISO ∈ [weekStartISO, weekEndISO]
  ```

迁移指南（从页面内复制逻辑到模块调用）
- 移除页面中的本地枢轴实现（isoDateWithPivot、completionLogicalDT 等）。
- 统一改为：P.logicalTodayISO、P.completionISO/DT、P.isLateByPivot（或使用 P.bind(3) 的便捷函数）。
- 当模块未加载时的最小化降级（可选）：
  - 可直接用自然日比较（toISODate、startOf('day')），此时不进行枢轴偏移。

注意事项
- 时区：模块内部统一为本地时区计算；跨时区字符串会被转为本地再比较。
- 日期类型：仅日期值不做枢轴；具体时间在 < 03:00 时才前移到上一逻辑日。
- 插件加载顺序：dynbedded 引入块需位于页面顶部，保证 window.ObLife.pivot3am 已就绪再调用。
- 刷新索引：修改模块后如页面分类不更新，刷新 Dataview 索引或切换视图以触发重算。

变更记录（与旧版的差异）
- 新增/强调：统一本地时区；对“仅日期”不应用枢轴偏移。
- 逾期判定改为比较枢轴后的 ISO 日期字符串（避免“同日到期且同日完成”误判为逾期）。
- 提供 bind(3) 便捷函数，减少页面样板代码。
