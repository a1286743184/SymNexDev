# LifeOS 插件化升级需求汇总

## 1. 背景与问题
- 现状为“散装形态”：多个 md 看板 + 多个 js 工具库 + 多个配置文件分散在 vault 中。
- 版本更新难以统一：旧入口脚本未替换会导致行为不一致，热修复不可控。
- 目标是产品化与可持续迭代，需要类似插件的一键更新能力。

## 2. 核心目标
- **体验几乎不变**：用户仍可通过 md 看板与配置页使用系统。
- **能力插件化**：真正的渲染、配置读写、索引与性能优化全部收敛到插件。
- **可更新可迁移**：统一版本入口、自动兼容与配置迁移。
- **双板块明确**：任务、财务两大板块分别有控制台与配置界面。

## 3. 用户体验要求
- 保留现有“打开 md 看板”的操作习惯。
- 任务与财务各有一个“控制台（Console）”作为更高层级入口。
- 控制台可作为 Obsidian 面板/弹窗快捷打开，也可通过 md 壳文件渲染。
- 看板与控制台 md 文件可配置默认生成目录。
- 用户拖动/改路径后，插件应支持按文件名重匹配并联动更新路径配置。

## 4. 插件功能需求
### 4.1 配置管理
- 插件设置界面分为“任务 / 财务”两大 Tab。
- 每个 Tab 结构接近现有独立配置 md 的视觉与内容（分区卡片、说明、列表编辑、保存/重载）。
- 配置存储在插件 data.json 作为唯一来源，支持从现有 vault 配置自动导入。
- 可选提供导出到 vault 的 JSON 以便备份与透明查看。

### 4.2 渲染与入口
- 以插件 CodeBlock 渲染替代 dataviewjs：
  - ```lifeos-task``` 渲染任务相关界面
  - ```lifeos-finance``` 渲染财务相关界面
  - ```lifeos-config``` 渲染配置界面（task/finance）
- md 看板与配置页保留为“壳文件”，内容仅调用插件渲染。

### 4.3 任务模块（Task）
- 数据来源：按现有“今日聚焦.config.json”的 sources 概念（目录/文件）。
- 任务解析：支持 `- [ ]` / `- [x]` 行、现有日期/时间标记格式、标签/胶囊规则。
- 过滤/分组/排序：复刻当前“今日聚焦/任务看板”逻辑与口径。
- 性能：任务索引缓存，避免全量重复扫描。

### 4.4 财务模块（Finance）
- 兼容现有 frontmatter 单账单文件结构与 CSV/ledger 扩展。
- 延续 6 分区配置体系：Sources / Schema / Normalization / Views / Budgets & Goals / Performance。
- ECharts 内置到插件（不再从 vault 加载外部脚本）。
- 月度/年度看板与预算/TopN/趋势等展示逻辑保持一致。
- 索引缓存与性能策略保留并由插件统一控制。

### 4.5 控制台（Console）
- **财务控制台**至少包含：
  - 创建指定年月的月度/年度看板
  - 打开财务配置
  - 索引/缓存管理
  - 资产落盘子系统入口与跳转（可先预留）
- **任务控制台**至少包含：
  - 打开任务配置
  - 索引/缓存管理
  - 打开核心子看板入口

### 4.6 创建看板能力
- 插件提供命令：创建月度/年度财务看板（支持选择年月与保存目录）。
- 可配置默认生成路径，且在路径变动后支持按文件名重新绑定。

### 4.7 更新机制（非官方商店）
- 插件按 Obsidian 标准结构交付。
- 通过离线更新脚本覆盖版本文件，保留用户配置文件：
  - 覆盖：manifest.json / main.js / styles.css / assets
  - 保留：data.json / 用户自定义设置与缓存
- 脚本需包含备份、白名单覆盖与最小权限策略。

## 5. 迁移与兼容要求
- 插件首次启动自动导入现有配置（任务/财务 JSON）。
- 旧的 dataviewjs 看板统一替换为插件 CodeBlock 壳。
- 原有功能口径保持一致，不改变业务逻辑与数据解释。
- 旧散装脚本文件可列出清单，由用户确认后清理。

## 6. 非功能性要求
- 可更新与可回滚（更新前备份）。
- 性能可控（缓存、增量扫描、最大扫描数量）。
- 透明与可追踪（日志、错误提示、诊断入口）。
- 跨平台可扩展（Windows 用 bat/ps1，后续可提供 macOS/Linux 方案）。

## 7. 验收要点
- 任务与财务看板数据与旧版一致。
- 控制台与配置页可从面板/命令与 md 壳两种方式访问。
- 更新插件不会覆盖用户配置。
- 拖动 md 看板位置后，能按文件名重新绑定并继续渲染。
