# SymbiNexus 独立应用化 -- 第二轮优化方案

> 汇总时间: 2026-04-14 | 基于第二轮3个交叉审核子代理的结果

---

## 一、核心决策确认

### 跨平台框架: Flutter 3.41+ (确认)

**加权评分: Flutter 8.15 vs Tauri 4.55**

关键审核结论:
1. Tauri Android端生态处于"早期采用者阶段"，tauri-plugin-widgets作者自述"尚未在真实项目中测试"
2. WebView渲染是Tauri的致命短板: 复杂看板30-45fps vs Flutter Skia 55-60fps
3. Dart 2周上手 vs Rust 2-3月上手，差距占MVP工期50%+
4. C盘差距优化后仅2-3GB，不应成为决策因素
5. 没有任何已上架Google Play的Tauri Android生产级应用

### 技术栈确认

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端框架 | Flutter 3.41+ | Dart语言，Skia/Impeller渲染 |
| 状态管理 | Riverpod 2.x | 编译时安全，依赖注入 |
| 路由 | GoRouter | 声明式路由，深链接支持 |
| 本地数据库 | SQLite + drift | 类型安全ORM，FTS5全文索引 |
| 文件解析 | yaml + markdown | YAML frontmatter解析 + Markdown渲染 |
| 图表 | fl_chart | 饼图/折线图/柱状图 |
| AI层 | dio(HTTP) + 本地规则引擎 | LLM API调用 + 离线降级 |
| 通知 | flutter_local_notification | 本地通知调度 |
| 后台任务 | workmanager | Android WorkManager封装 |
| 小组件 | home_widget | Android桌面小组件 |
| 同步 | Syncthing REST API + 文件监听 | 外部Syncthing进程管理 |
| 中文分词 | unicode61(MVP) -> jieba-rs via FFI(正式版) | 渐进增强 |
| 安全存储 | flutter_secure_storage | API密钥加密存储 |

## 二、第一轮方案的关键修正

### 修正1: MVP工期 10-14周 -> 14-18周

原因:
- 9个JS工具库(~3000+行)需重写为Flutter原生组件
- Templater自动触发机制需设计替代方案
- frontmatter格式不一致需全量扫描和容错处理
- jieba中文分词Dart实现不成熟，需改用unicode61+后续FFI方案

### 修正2: C盘占用预期

| 框架 | 默认C盘 | 优化后C盘 | 优化后D盘 |
|------|---------|----------|----------|
| Flutter | 23-58GB | 5-8GB | 15-25GB |
| Tauri | 14-30GB | 3-8GB | 10-20GB |

差距仅2-3GB，不应影响选型。

### 修正3: 冲突自动合并策略

MVP阶段不应尝试自动合并，仅检测+提示用户。原因:
- Markdown frontmatter存在格式不一致(引号/无引号)
- Syncthing-Fork跨App通信在Android 11+可能受限
- 自动合并风险高，可能导致数据丢失

### 修正4: 中文分词方案

- MVP: unicode61 tokenizer（基础分词，覆盖大部分场景）
- 正式版: jieba-rs via FFI（精确中文分词）
- 不使用jieba-dart（实现不成熟）

- 02-复盘系统/01-日复盘/ -> 日复盘模板

- 01-经纬矩阵系统/02-周委托模块/ -> 周计划模板

- 05-生活坐标系统/03-装备档案/ -> 装备档案模板

替代方案: 新应用中设计"文件创建事件监听 + 模板引擎"

当前方案: PowerShell脚本 + ntfy.sh推送通知

- 写回功能(标记已提醒)

## 四、3个严重问题及解决方案

### 问题1: frontmatter格式不一致

解决方案:
1. 迁移前运行全量扫描脚本，检测格式差异
2. 新应用的YAML解析器使用容错模式
3. 写入时统一为无引号格式（与最新文件一致）

### 问题2: Syncthing-Fork跨App通信

解决方案:
1. MVP阶段以文件系统监听(watchdog)为主
2. REST API作为辅助监控手段
3. Android 11+使用MediaStore API或SAF访问共享文件
4. 长期方案: 嵌入Syncthing二进制

### 问题3: JDK版本冲突

解决方案:
1. 安装JDK 17到D盘（从adoptium.net下载zip版）
2. JAVA_HOME指向JDK 17
3. 保留JDK 20用于其他项目
4. Android Gradle Plugin 8.x要求JDK 17

## 五、优化后的MVP范围

1. 基础框架（Flutter + 路由 + 主题 + SQLite/drift）
